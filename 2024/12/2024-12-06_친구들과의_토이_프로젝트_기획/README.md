# 친구들과의 토이 프로젝트 기획

최근 친구들과 JPA를 활용한 토이 프로젝트를 기획하고 있다.  
친구 중 한 명은 [2024-11-29 친구와 한잔](/2024/11/2024-11-29_친구와_한잔/README.md)에서 언급했던, **고도화 프로젝트에서 MyBatis를 JPA로 전환**하며 새롭게 JPA를 공부 중인 친구다.  
또 다른 친구는 대학교 졸업 후 IT회사에서 근무하다가 장기간의 공백기를 가진 후, 최근 다시 취업을 준비하며 JPA에 관심을 두고 있다.

이런 상황에서 JPA를 극한으로 활용해보고, 서로의 역량을 키울 수 있는 토이 프로젝트를 기획하고 있다.  
프로젝트의 목표는 **실무에서 JPA를 효과적으로 사용하는 방법을 배우고, 각자의 성장을 도모**하는 것이다.

---

## 프로젝트 후보

현재 두 가지 프로젝트 아이디어를 고민 중이다.

### **1. 블로그 + 채팅 서비스**
블로그 서비스는 복잡한 도메인 모델링과 다양한 조회 요구사항을 구현하는 데 적합하며, 채팅 기능을 추가해 실시간 데이터 처리를 경험할 수 있다.

**핵심 기능:**
- **도메인 모델링:**  
  User, Post, Comment, Tag, Category, Like 등 다양한 관계를 포함한 복합 도메인 모델 설계.  
  - 다대일(One-to-Many), 일대다(Many-to-One), 다대다(Many-to-Many) 관계를 활용.
  
- **다양한 조회 쿼리:**  
  QueryDSL을 활용해 동적 필터링, 정렬, 페이징 처리를 구현.  
  - 예: 특정 태그를 포함한 게시글 필터링, 좋아요 수로 정렬된 게시글 조회.

- **캐시 사용:**  
  Redis를 활용해 인기 게시글을 캐싱하고, 데이터 접근 성능 최적화.

- **엔티티 생명주기 활용:**  
  - 게시글 작성 시 @PrePersist를 사용해 자동으로 태그를 생성.  
  - 수정 시 @PreUpdate를 활용해 수정 이력을 관리.

- **지연 로딩(Lazy)과 즉시 로딩(Eager) 테스트:**  
  두 로딩 전략의 성능 차이를 상황별로 비교하고 이해.

---

### **2. 소셜 네트워크 서비스**
소셜 네트워크 서비스는 대용량 데이터 처리와 비동기 이벤트 처리를 경험할 수 있는 프로젝트로, 실무에서 자주 접하는 문제를 해결하는 데 적합하다.

**핵심 기능:**
- **팔로워/팔로잉 관계:**  
  유저 간 다대다(Many-to-Many) 관계를 구현.

- **피드 기능:**  
  유저의 팔로워 활동에 따라 동적으로 구성되는 피드 데이터 처리.

- **대용량 데이터 관리:**  
  - 피드 조회 성능 최적화.  
  - Redis 등을 활용한 캐싱 전략.

- **Event Listener 활용:**  
  - 알림 전송 등 특정 이벤트 발생 시 비동기 작업 처리.

## 진행 방식

### **1. 규칙 정하기**  
가장 중요한 것은 각자의 일정과 작업 속도를 고려하여 **꾸준히 개발하고 리뷰하는 규칙을 세우는 것**이다.  
- **주 단위 목표 설정:** 매주 각자가 맡은 기능을 개발하고, 이를 리뷰하는 시간을 갖는다.  
- **리뷰 중심 협업:** 코드 리뷰를 통해 서로의 코드를 학습하고 개선 방향을 논의한다.  
- **진행 상황 공유:** 프로젝트 진행 중 발생한 문제나 고민을 정리하고, 함께 해결 방안을 모색한다.

### **2. 학습과 성장 목표**  
- JPA의 다양한 기능을 실무적으로 경험하며, 각자 부족한 부분을 보완한다.  
- 프로젝트를 통해 **협업 능력과 도메인 설계 역량**을 키운다.  
- **소프트 스킬:** 리뷰와 피드백을 통해 서로 성장할 수 있는 협력 문화를 만든다.

## 결론

이번 토이 프로젝트는 기술을 배우는 것은 기본으로 협업과 학습의 장으로 삼고자 한다.  
각자가 겪고 있는 상황과 관심사에서 출발해, JPA를 극한으로 활용할 수 있는 실전 프로젝트를 설계 해보고자 한다.  
블로그 + 채팅 서비스와 소셜 네트워크 서비스 중 어느 방향이든, **꾸준히 협업하며 성장**할 수 있는 경험이 되리라 생각한다.
